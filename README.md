# Метод адаптивної ідентифікації елементів веб-інтерфейсів в умовах динамічної модифікації DOM-структури

**Дипломна робота** — Ярослав, 2026

## Опис проекту

Система автоматично відновлює виконання Selenium-тестів коли DOM-структура сторінки змінюється (перейменування ID, класів тощо), але візуальне відображення залишається незмінним.

**Проблема:** Класичні Selenium-тести покладаються на CSS-селектори та ID елементів. Будь-який рефакторинг верстки призводить до `NoSuchElementException` і падіння тестів — навіть якщо сторінка виглядає і працює коректно.

**Рішення:** При виникненні помилки система робить скріншот сторінки і запускає YOLOv8 для візуального пошуку елемента. Модель визначає координати елемента (кнопки, поля вводу) і тест продовжується без жодних змін у тестовому коді.

## Як це працює

### 1. Звичайний режим (v1)
Selenium знаходить елементи за стандартними ID (`email`, `password`, `login-btn`). Self-healing не активується.

### 2. Режим відновлення (v2)
```
Selenium: find_element(By.ID, "login-btn")
→ NoSuchElementException (ID перейменовано на "submit-action")
→ SelfHealingDriver перехоплює помилку
→ Робить скріншот сторінки
→ YOLOv8 детектує елемент класу "Button" з впевненістю 0.83
→ document.elementFromPoint(630, 431) повертає DOM-елемент
→ Тест продовжується ✅
```

### 3. Класи детекції
| Клас | Елемент |
|------|---------|
| `Button` | Кнопки (`<button>`, `<input type="submit">`) |
| `input` | Поля вводу (`<input type="text/email/password">`) |
| `Dropdown` | Випадаючі списки (`<select>`) |
